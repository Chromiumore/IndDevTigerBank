# Кратко опишите предметную область: что моделирует программа
Программа моделирует систему учёта финансов банка.

# Используйте единый code style
 * Названия классов в PascalCase
 * Названия абстрактных классов содержат в себе слово "Base"
(избегается использование слов "Abstract" в названии абстрактных классов,
использования названий по типу IService для интерфейсов, ServiceImp для
реализаций интерфейсов)

# Соблюдайте принципы SOLID
## Single Responsibility
Каждый сервис отвечает за свою доменную область
## Open/Closed
Стратегии экспорта позволяют добавлять новые форматы
## Liskov Substitution
Все наследники BaseRepository могут использоваться везде, где ожидается BaseRepository
## Interface Segregation
Репозитории содержат только нужные им методы
## Dependency Inversion
Родительский класс репозиториев зависит от абстрактного класса сущности, а не от
конкретной реализации

# Реализуйте взаимодействие модулей через DI-контейнер
Spring управляет жизненным циклом объектов и их зависимостями.
Это упрощает тестирование и делает связи между классами явными.
Пример: Внедрение зависимостей через конструктор класса AnalyticsService
